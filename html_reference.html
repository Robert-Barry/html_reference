HTML Reference

A website that provides a reference for HTML elements. It does not include any CSS so that the elements can be seen exactly as the browser renders them.

The Document Object Model or DOM

A tree structure of HTML objects created by the web browser. Each object in the Dom tree is called a node.

Tags and Attributes

Keywords between <> are called HTML tags and are used to describe the content.

HTML attributes offer additional information about the content of the HTMLL tags and are formatted as name-value pairs, such as name="value".

HTML comments

HTML comments are created using the following code:

<!-- This is a comment -->

HTML Document Framework

The following is a simplified HTML framework for creating an HTML document.

<!doctype html>
<html>
  <head>
    <meta charset="UTF-8">
    <title>Page Title</title>
  </head>
  <body>
    <!-- visible content -->
  </body>
</html>

HTML Document Type

<!doctype html> - Must be the first declaration of an HTML document before <html>.
A document type isn't an HTML tag, but instructs the webpage which HTML version is being used.

The <html> element

<html></html> - The root element of every HTML file. Informs the browser that this is an HTML document.
Only <head> and <body> are allowed as children of the <html> tag.

The <head> element

<head> - Elements between the head tags are not displayed in the web browser, but provides information specific
to the browser and to search engines, inbcluding elements that enable control of the content and display of the web page.

<head> elements

<title>...</title> - Must be included. Contains the title of the HTML document.

<base> - Sets the base URLs/targest for all relative URLs in a web page. href and target attributes are required.

Attributes:

  • href - defines the base URL. Used by the web rowser as the base address for relative or absolute paths.
  • target - Specifies the target window in which the link target should be displayed: _self, _blank, _parent, _top

<link> - Sets logical links of the HTML document to other files to be included.

Attributes:

  • href - Specifies the URL to the resource to be linked.
  • hreflang - The language of the resource to be linked.
  • media - The medium or device for which the target resource in href has been optimized.
  • rel - The relation of the external resource to the current document
    • alternate - Links to an alternative presentation form of the current page.
    • author - Links to another page with info about the author of the current document.
    • archives - Links to a previous version of certain documents.
    • help - Links to a help document.
    • icon - Allows the assignment of a favivon to the web page.
    • license - Links to the usage rights for the content of the current page.
    • next, prev - Creates a link from the current page to the next or previous page.
    • prefetch - Links to an external web page that is loaded into cache before hand.
    • pingback - Specifies the website of a pingback server.
    • search - Links the current document to another document where a search across the whole website is possible.
    • stylesheet - Links to an external CSS file.
    • tag - A linked resource that applies to the current document.
  • size - Specifies the size of the resouce to be linked.
  • type - Speicifies the MIME type for the document to link to.

<style> - Sets the local stylesheet rules for the HTML document.

Attributes:

  • media - Which medium/device the target resource in href has been opyimized for.
  • type - Specifies the MIME type for the stylesheet.

<script> - Integrates the client-side scripts. Must have an opening and closing <script> tag, even when linking to an external document.
    Example: <script type="text/javascript" src="script.js"></script>

Attributes:

  • async - Execute the script asynchronusly with the HTML document. External scripts only.
  • charset - Sets the encoding for the external script.
  • defer - Parses the HTML first before the script. External scripts only.
  • src - Specifies the URL to the external script.
  • type - Specifies the MIME type of the script, usually text/javascript.

<meta> - Sets the metadata such as keywords, descriptions, or the character set for the HTML document.

Attributes:

  • name/content - A name value pair of any information: <meta name="author" content="John Doe">
  • http-equiv/content - Intended for communication with a web browser: <meta http-equiv="refresh" content="5">
  • charset - Specifies the character encoding.

Viewport - <meta name="viewport" content="width=device-width, initial-scale=1.0, shrink-to-fit=no"> - tells the browser to use the width of the device. Used to adapt to different device sizes.`

The <body> element

<body></body> - The visible content of an HTML file.

Structural Elements

The following elements serve the purpose of dividing the content into relativant semantic sections. There is no visible indication when used.

  • <section> - Subdivision of the HTML document into different sections.
  • <article> - Subdivirion of content into a self-contained block based on topic.
  • <aside> - Content such as a sidebar or additional information about the main content.
  • <nav> - Makrks up navigation.
  • <h1> - <h6> - Headings from 1st to 6th order.
  • <header> - The header of the content.
  • <footer> - The footer of the content.
  • <address> - Contact information such as a street address.

Elements for Structuring Text

<p> - Paragraph tag. Common for displaying text.

<br> - A line break in the text.
For example: <p>Hello<br>there!</p>
Appears as:

Hello
there!

<wbr> - Optional line break within a word. (alternatively use &shy;)

<hr> - Topic based separation at the paragraph level.
For example: <p>Topic 1<hr>Topic 2</p>
Appears as:

Topic 1

Topic 2

<blockquote> - Citation as a text paragraph.
For example: <blockquote>Lorem ipsum dolor sit amet</blockquote>
Appears as:

Lorem ipsum dolor sit amet.

Attributes

  • cite - Specifies the source of the citation: <blockquote cite="www.fake-quotes.com">

<div> - Defines a general section of the document.

<main> - The main content of the web page. Occurs only once in a document. Not a section element, only a grouping eleemnt.

Attributes

  • hidden - Hides the main element and allows for multiple <main> elements as long as only 1 appears at a time.

<figure> - Grouping or summarizing content for separate description.

<figcaption> - Labeling content grouped via the figure element.

<ul> - Creates and unordered list.

<ol> - Creates an ordered list.

Attributes

  • reversed - reverses the number ordering of an ordered list: <ol reversed="reversed">.
  • start - Changes the starting numbe rof an ordered list: <ol start="3">
  • value - Used in the <li> element of an ordered list to change the number of a list item: <li value="5">

<li> - A list element. Part of <ul> and <ol>.

<dl> - The root elemnt for a description list.

<dt> - The term being described.

<dd> - The description of the term.

Text Markup

  • <abbr> - Abbreviations or acronyms. Example: <abbr title="world wide web">WWW</abbr>
  • <cite;> - Where the source is from. Used to mark up the working title of a book, movie, painting, etc.
  • <code> - Computer code within a paragraph.
  • <pre> - Preformatted text deisplaying all spaces and line breaks.
  • <kbd> - Keyboard input.
  • <samp> - Screen output of a program.
  • <dfn> - Defining a term. Should mark-up the term itself.
  • <var> - A variable.
  • <bdo> - Change the text direction for bidirectional text.
  • <bdi> - Defining a section for bidirectional text.
  • <em> - Highlighting text that would be emphasized in spoken language.
  • <strong> - Highlighting important words and passages.
  • <i> - Words that technical terms, thoughts, or foreign words.
  • <b> - Meningful names or keywords.
  • <mark> - Highlighting text with a marker.
  • <q> - Words or passages as cited or spoken text.
  • <u> - Text underlined as a proper name or incorrect words or passages.
  • <s> - Text that is no longer valid.
  • <ins> - Newly added or revised text.
  • <del> - Deleted text.
  • <sub> - Subscript.
  • <sup> - Superscript.
  • <time> - Dates and times.Use the datetime attirubte for machine readable date and time: <time datetime="2025-12-20>December 20, 2025</time>
  • <small> - Small print.
  • <ruby>, <rp>, <rt> - Ruby annotations.
  • <span> - General section with a paragraph of text.

Graphics & Multimedia

  • <img> - A graphic file in HTML.
  • <map>, <area> - A link-sensitive graphic.
  • <picture>, <source> - Load an image from several image sources.
  • <svg> - A scalable vector graphic.
  • <math> - A formula written with MathML.
  • <canvas> - A canvas used for drawing graphics.
  • <video> - Video files without plugins.
  • <audio> - Audio files without plugins.
  • <embed>, <object>, <iframe> - A way to embed active elements.

<img> Element

The <img> element inserts a graophic at the position in which the <img> element is placed. The attributes src and alt must be present.
<figure> and <figcaption> are often used with images by enclosing them in <figure>.

Attributes

  • src - The source of the image file.
  • alt - An alternative description of an image file.
  • title - A global attribute that sometimes makes sens to include, such as if the image is a link.
  • height - Used to specify the height of an image.
  • width - Used to specify the width of an image.
  • ismap - Used if the images are server-side image maps.
  • usemap - Specifies the name of the image map to be associated with the image.

Example

        
    <p>
        <img src="images/example.jpg" alt="An example image">
        <img src="images/example2.jpg" alt="Another example image">
    <p>

Image Maps

Image maps are just links embedded within a graphic. It's a way to make a graphic have multiple links in one image.

  • <img> - The graphic in a map element with an anchor.
  • <map> - The area for the link-sensitive graphic.
  • <area> - Defines a clickable area in an image.

Example

        
    <p>
        <img src="images/image.jpg" alt="A linked image" usemap="#map">
    </p>
    <map name="map">
        <area shape="rect" coords="0, 0, 100, 100" href="exmaple.html" alt="example" title="Example">
        <area shape="rect" coords="0, 100, 100, 100" href="exmaple2.html" alt="example2" title="Example2">
    </map>

<picture>

Provides a suitable image for all display sizes.

Example

        
    <picture>
        <source media="(min-width: 1024px)" srcset="images/example-1024.jpg">
        <source media="(min-width: 640px)" srcset="images/example-640.jpg">
        <source media="(min-width: 480px)" srcset="images/example-480.jpg">
        <!-- Fallback for older browsers -->
        <img src="images/example-480.jpg> alt="An example image">
    </picture>

Attributes

  • srcset - The media source to be loaded.
  • media - The media query for the source to be loaded. Can only be used in the <picture> element.
  • type - The MIME type to which the linked file belongs.
  • sizes - The width or pixel density at which the image should be displayed.

Favicons

Added to the site through the <link> element in the header with the attribute rel="icon".
The size is ususally 16 x 16, sometimes 32 x 32. The images should be saved with the .ico format.

Example

        
    <link rel="icon" type="image/png" sizes="16x16" href="imaghes/favicon.ico">
    <!-- For mobile devices -->
    <link rel="apple-touch-icon" type="image/png" sizes="180x180" href="imaghes/favicon180.png">

Vector Graphics

Use scalable vector graphics (SVG) to display vector images. It can be added using the <imhage> tag.

Example

        
    <p>
        <img src="vector/vector.svg" width="100" height="100" alt="A vector image">
    </p>

SVG can also be directlt embedded into the webpage. The child elements of SVG must use XML rules, including a closing / on empty elements.

        
    <p>
        <svg width="300" height="100">
            <ellipse ry="45" rx="140" cy="50" cx="150" fill="red" stroke-width="10" strok="black">
            <text text-anchor="middle" font-size="60" y="67" x="150" fill="black">SVG</text>
        </svg>
    </p>
