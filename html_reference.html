HTML Reference

A website that provides a reference for HTML elements. It does not include any CSS so that the elements can be seen exactly as the browser renders them.

The Document Object Model or DOM

A tree structure of HTML objects created by the web browser. Each object in the Dom tree is called a node.

Tags and Attributes

Keywords between <> are called HTML tags and are used to describe the content.

HTML attributes offer additional information about the content of the HTMLL tags and are formatted as name-value pairs, such as name="value".

HTML comments

HTML comments are created using the following code:

<!-- This is a comment -->

HTML Document Framework

The following is a simplified HTML framework for creating an HTML document.

<!doctype html>
<html>
  <head>
    <meta charset="UTF-8">
    <title>Page Title</title>
  </head>
  <body>
    <!-- visible content -->
  </body>
</html>

HTML Document Type

<!doctype html> - Must be the first declaration of an HTML document before <html>.
A document type isn't an HTML tag, but instructs the webpage which HTML version is being used.

The <html> element

<html></html> - The root element of every HTML file. Informs the browser that this is an HTML document.
Only <head> and <body> are allowed as children of the <html> tag.

The <head> element

<head> - Elements between the head tags are not displayed in the web browser, but provides information specific
to the browser and to search engines, inbcluding elements that enable control of the content and display of the web page.

<head> elements

<title>...</title> - Must be included. Contains the title of the HTML document.

<base> - Sets the base URLs/targest for all relative URLs in a web page. href and target attributes are required.

Attributes:

  • href - defines the base URL. Used by the web rowser as the base address for relative or absolute paths.
  • target - Specifies the target window in which the link target should be displayed: _self, _blank, _parent, _top

<link> - Sets logical links of the HTML document to other files to be included.

Attributes:

  • href - Specifies the URL to the resource to be linked.
  • hreflang - The language of the resource to be linked.
  • media - The medium or device for which the target resource in href has been optimized.
  • rel - The relation of the external resource to the current document
    • alternate - Links to an alternative presentation form of the current page.
    • author - Links to another page with info about the author of the current document.
    • archives - Links to a previous version of certain documents.
    • help - Links to a help document.
    • icon - Allows the assignment of a favivon to the web page.
    • license - Links to the usage rights for the content of the current page.
    • next, prev - Creates a link from the current page to the next or previous page.
    • prefetch - Links to an external web page that is loaded into cache before hand.
    • pingback - Specifies the website of a pingback server.
    • search - Links the current document to another document where a search across the whole website is possible.
    • stylesheet - Links to an external CSS file.
    • tag - A linked resource that applies to the current document.
  • size - Specifies the size of the resouce to be linked.
  • type - Speicifies the MIME type for the document to link to.

<style> - Sets the local stylesheet rules for the HTML document.

Attributes:

  • media - Which medium/device the target resource in href has been opyimized for.
  • type - Specifies the MIME type for the stylesheet.

<script> - Integrates the client-side scripts. Must have an opening and closing <script> tag, even when linking to an external document.
    Example: <script type="text/javascript" src="script.js"></script>

Attributes:

  • async - Execute the script asynchronusly with the HTML document. External scripts only.
  • charset - Sets the encoding for the external script.
  • defer - Parses the HTML first before the script. External scripts only.
  • src - Specifies the URL to the external script.
  • type - Specifies the MIME type of the script, usually text/javascript.

<meta> - Sets the metadata such as keywords, descriptions, or the character set for the HTML document.

Attributes:

  • name/content - A name value pair of any information: <meta name="author" content="John Doe">
  • http-equiv/content - Intended for communication with a web browser: <meta http-equiv="refresh" content="5">
  • charset - Specifies the character encoding.

Viewport - <meta name="viewport" content="width=device-width, initial-scale=1.0, shrink-to-fit=no"> - tells the browser to use the width of the device. Used to adapt to different device sizes.`

The <body> element

<body></body> - The visible content of an HTML file.

Structural Elements

The following elements serve the purpose of dividing the content into relativant semantic sections. There is no visible indication when used.

  • <section> - Subdivision of the HTML document into different sections.
  • <article> - Subdivirion of content into a self-contained block based on topic.
  • <aside> - Content such as a sidebar or additional information about the main content.
  • <nav> - Makrks up navigation.
  • <h1> - <h6> - Headings from 1st to 6th order.
  • <header> - The header of the content.
  • <footer> - The footer of the content.
  • <address> - Contact information such as a street address.
